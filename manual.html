iTrajComp: interactive Trajectory Comparison

(former RMSDTT2)

Description

iTrajComp is a VMD plugin for general analysis of trajectories. It is under heavy development, but it can already do a few interesting things (no guarantees, though!). In hopes of helping our fellow VMD users, this webpage will host the preview releases of the tool. If you are interested in trying this tool, please contact the author. Once we are confident that the plugin is doing what it should and we get some time to write a little manual, it will be freely available from this webpage.

Note: This plugin is the former RMSDTT2 plugin. We decided to change the name a) to avoid confusion with the RMSDTT plugin, and b) because it is designed to do much more than simple calcuating the rmsd.

Features

Very briefly, the features of the plugin:

  • Calculations:
    • Based on frames (molecules and frames):
      • contacts
      • hbonds
      • labels
      • rgyr
      • rmsd
    • Based on segments (residues, atoms, ...):
      • covar
      • dist
    • RMSD:
      • coordinates (traj(1) vs [traj(1) or traj(2)])
      • labels (distance, angles, dihedrals)
    • Number of:
      • contacts within a cutoff
      • hydrogen bonds within a cutoff, angle (as defined in VMD)
  • Objects:
    • matrix representation
    • interaction with VMD main window (structures involved in the data clicked will be represented in VMD)
    • multiple simultaneous objects
    • combination of objects in a new object
    • save the data in 4 file formats

Progress bar. Can slow down the calculations, but allows for interactivity of the GUI.

Calculation modes

iTrajComp supports two calculation modes: frames and segments.

Frames

In frames mode calculations are based on molecules and frames. Each cell in the graph represents a pair of molecule/frame. For exapmle, cell (0 1) (3 5) in a rmsd graph contains the rmsd between (molecule 0, frame 1) and (molecule 3, frame5).

In this mode the number of cells in the graph will depend on the number of molecules and frames, and is independent of the number of atoms. For example, a rmsd calculation with 2 molecules, each with 20 frames, 25 residues and 100 atoms will results in a graph of 40 by 40 cells.

Segments

In segments mode calculations are based on molecule segments, ie, residues or atoms. Each cell in the graph represents a pair of segments. For example, cell (5 Glu) (7 Arg) in a dist by residue graph contains the distance between residues 5 (Glu) and 7 (Arg).

In this mode the number of cells in the graph will depend on the number of segments (atoms or residues) and is independent of the number of molecules or frames. For example, a dist calculation with 2 molecules, each with 20 frames, 25 residues and 100 atoms will results in a graph of 25 by 25 cells if the segments are residues and 100 by 100 if the segments are atoms.

Data types

To support diferent calculations, from VMD and user defined, iTrajComp recognizes several data types sets and formats.

Sets

Data type sets are used to work on data that contain a single set of values, dual set of values or multiple sets of values.
  • Single sets hold one set of values of the same format, ie. the radius of gyration calculation type (rgyr) gives a single value for each cell.
  • Dual sets can hold 2 types of sets, where the first set represents single values to be plotted on the graph, and the second set holds extra information for each cell. The second set can be used in the representations or for reference when. It can be accessed when clicking in a cell with the middle mouse button. For example, the hydrogen bonds calculation type (hbonds) holds the number of hydrogen bonds in the first set and the information about each hydrogen bond (donnor, acceptor and hydrogen indexes) in the second set.
  • Multiple sets hold the raw data that is used to create a single value for each cell. This is better explained with an example. The distance calculation type (dist) is used to represent the distance between the same atom or residue in the different conformations (or trajectory frames). The value shown in the graph by default is the average of the distance among the conformations. However other statitical measures, like standard deviation, minimum and maximum can also be represented. These values are the multiple sets. The individual values for each pair of conformations can be found for each cell clicking the middle mouse button.

Formats

Calculation types

Summary of calculation types
Type Description Mode Data type
rmsd Rmsd in coordinate space frames single / dual
rgyr Rmsd of the radius of gyration frames single
labels Rmsd of bonds, angles or dihedrals frames dual
contacts Number and list of contacts frames dual
hbonds Number and list of hydrogen bonds frames dual
dist Distance between segments segments multiple
covar Covariance between segments segments single

Rmsd in coordinate space (rmsd)

Calculates the rmsd among conformations.

Rmsd of the radius of gyration (rgyr)

Calculates the rmsd of the radius of gyration among conformations.

Rmsd of bonds, angles or dihedrals (labels)

Calculates the rmsd of bonds, angles or dihedrals among conformations.

Number and list of contacts (contacts)

Calculates the number of contacts among conformations and also returns a list of pairs of contacts.

Number and list of hydrogen bonds (hbonds)

Calculates the number of hydrogen bonds among conformations and also returns a list of hydrogen bonds.

Distance between segments (dist)

Calculates the distance amongs segments.

Covariance between segments (covar)

Calculates the covariance amongs segments, ie the covariance matrix.
